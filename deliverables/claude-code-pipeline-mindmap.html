Orchestration — How Claude Code Runs This
The execution model. This is where the blind spots live.
Execution Model
  • You: "Process the next 50 leads for Owner.com"
  • Claude Code: reads DB → loops through leads → calls APIs → writes results back
  • Single Python/Node script orchestrates everything
  • Progress saved to DB after each lead (checkpoint)
The Script Architecture
  • process_leads.py — main orchestrator
  • Reads config: client name, batch size, approach files
  • Connects to DB, pulls pending leads
  • For each lead: runs pipeline steps sequentially
  • Saves results after EACH lead (crash-safe)
  • Logs everything to file + DB
Blind Spot No Auto-Scheduling
  • Claude Code doesn't run by itself
  • Someone must say "process the leads"
  • No cron, no trigger, no webhook listener
  • Mitigation A: Claude Code scheduled tasks MCP
  • Mitigation B: cron job that runs the Python script directly
  • Mitigation C: Accept manual trigger (fine for weekly batches)
Blind Spot Sequential Processing
  • Claude Code processes one lead at a time
  • ~30-60 seconds per lead (API calls + LLM inference)
  • 50 leads ≈ 30 minutes
  • 500 leads ≈ 5 hours
  • Mitigation: Python script with asyncio can parallelize API calls (3-5 concurrent leads)
  • Mitigation: Process in batches across sessions
Blind Spot Session Limits
  • Claude Code sessions have token/time limits
  • Long-running processing may hit context limits
  • Mitigation: Build as standalone script, not conversational
  • Claude Code WRITES the script, then you RUN it
  • Script runs independently of Claude Code session
Blind Spot Error Recovery
  • What if Jina is down? API rate-limited? DeepSeek 500 error?
  • Need retry logic with exponential backoff
  • Need per-lead error status in DB
  • Need graceful degradation (skip research if SerpAPI fails)
  • Mitigation: Build into the Python script

What You Gain vs What You Lose
Honest assessment of the trade-offs
Gain Zero SaaS Costs
  • No Clay subscription for AI pipeline
  • No n8n hosting
  • Pay only for API calls (pennies)
Gain Full Control
  • Every prompt version-controlled in Git
  • Change anything instantly
  • No vendor lock-in
  • Debug any step by reading the script
Gain Single Tool
  • No Clay ↔ n8n ↔ API handoffs
  • No data sync issues
  • One place to debug
  • Claude Code writes + runs everything
Gain Better Model Access
  • Claude Sonnet/Opus for copywriting (vs Clay's GPT-4o-mini)
  • Direct API = latest models immediately
  • No waiting for Clay to add new models
Lose Visual Workflow Editor
  • No drag-and-drop like Clay or n8n
  • Pipeline is code, not visual nodes
  • Harder for non-technical team members
  • Trade-off worth it if you're the only operator
Lose Auto-Scheduling
  • No "run every Monday at 9am" built in
  • Need cron or scheduled tasks workaround
  • Acceptable if you run weekly batches manually
Lose Monitoring Dashboard
  • No visual execution logs like n8n
  • No error alerts
  • Mitigate with logging to DB + simple dashboard
Lose Parallel Processing
  • Sequential by default (1 lead at a time)
  • Mitigate with asyncio in Python (3-5x speedup)
  • Still slower than Clay's parallel column processing

The Key Architectural Decision
Script-based vs conversational execution
Option A: Claude Code WRITES the script
  • Claude Code creates process_leads.py
  • Script runs independently (python process_leads.py)
  • Can be cron'd, asyncio'd, parallelized
  • Claude Code iterates on it when you need changes
  • Session limits don't matter — script runs standalone
  • Best for: production, large batches, automation
Option B: Claude Code IS the runtime
  • You say: "Process 50 leads for Owner.com"
  • Claude Code runs the pipeline in-session
  • More interactive, can adjust on the fly
  • Limited by session duration / context window
  • Good for small batches and testing
  • Best for: testing, small lists, ad-hoc runs
Recommendation: Start with B, graduate to A
  • Use Option B to test and validate the pipeline with 5-10 leads
  • Once the flow works, have Claude Code package it into a standalone script (Option A)
  • Run the script via cron or manually for production batches
  • Come back to Claude Code when you need to modify the pipeline

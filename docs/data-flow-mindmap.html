Changes from Previous Architecture

Removed

  • BQRS Agent column + cost tracking column
  • {{booleansearches}} input to Researcher
  • {{researchreportstructure}} input to Researcher
  • Prescriptive paragraph-by-paragraph PATTERN format in approaches
  • "Follow the selected approach pattern exactly" instruction in Copywriter
  • DATA CONDITIONAL LOGIC section from approach files — copywriter handles data availability dynamically
  • Random approach assignment (replaced with Priority Cascade)
  • "Exact phrasing for opener" instruction in research report — researcher provides raw material, copywriter decides phrasing
  • "Hook Context" section from research report — replaced by approach-specific structures
  • Subject Line Templates section from approach files — subject lines handled outside Clay waterfall
  • Subject Line Picker formula column — removed entirely
  • Boolean Query Strategy from approach files — replaced with signal list (What Researcher Should Look For)

Added

  • Website Scrape column — dedicated Claygent that always runs (every row), provides {{companyProductsServices}} to Copywriter regardless of approach
  • Researcher self-directs query construction from approach (STEP 1)
  • Priority Cascade Formula for approach selection — data-informed, checks per-approach data availability
  • Data Handling instruction in Copywriter — adapts to missing variables dynamically, uses uncertainty language for proxy-based estimates
  • Company Context section in Copywriter — {{companyProductsServices}} always available, even for Math approach
  • Email 2 output from Copywriter (Mode A: expand/reframe, Mode B: 3 ideas)
  • Clay architecture: approach files are self-contained instruction sets — Email 2 mode includes full instructions, not just mode name
  • Researcher uses signal list from approach (no Boolean query syntax) — Claygent handles its own searching
  • Style, Tone & Spam Checker replaces old Flagged Word Replacer
  • {{LeadListContext}} added as input to Researcher
  • {{valueprop}} added as input to Style/Tone/Spam Checker
  • Data Variables list as skill output
  • Research Required auto-YES when Email 2 Mode = "3 Ideas"

Updated

  • Approach Selector → Priority Cascade formula (data-informed instead of random)
  • Approach PATTERN format → GOAL / SIGNAL / CONSTRAINT (strategic intent, not paragraph templates)
  • Copywriter → strategic brief mode + dynamic data handling + Company Context section (always available)
  • Researcher → website scrape separated into own column; researcher now focuses purely on approach-specific research
  • Research report → factual material only (Key Finding, Insider Remark Material as raw details, no suggested copy; no Company Products/Services — that's in the separate scrape)
  • Subject lines → removed from system entirely (handled outside Clay waterfall)
  • Approach files → self-contained for Clay columns (Email 2 full instructions, no Boolean syntax, Clay architecture note)
  • Copywriter prompt → "medium prompt" with universal rules only; strategic instructions come from the approach
  • Spam word list reduced from 300+ to ~100 words
  • Flagged Word Replacer → Style, Tone & Spam Checker (broader scope)

Net Cost Impact

Removed: BQRS Agent (saved: ~1 Claygent call per row where research = yes)
Added: Website Scrape column (new Claygent call, every row — lightweight, 50-100 word output)
Added: Email 2 from Copywriter (marginal token increase in existing call)
Subject lines: Removed from system — handled outside Clay waterfall
Approach Selector: Formula-based cascade, $0 per row (replaces $0 randomizer — same cost)
Net: BQRS removal saves 1 Claygent call on research rows. Website Scrape adds 1 lightweight Claygent call on ALL rows. For rows where research runs, this is roughly cost-neutral. For Math/no-research rows, there is a small new cost — but the copywriter now has company context it previously lacked, improving email quality.
